# 11 — Mark and manage Skill Pages

**What to build:** Let a user turn an existing or new Page into a reusable Skill while continuing to organize and edit it like any other Page in the workspace.

**Blocked by:** 05 — Isolate editor session and migrate autosave.

**Status:** ready-for-agent

- [ ] A user can mark an owned Page as a Skill without moving, copying, or replacing its document.
- [ ] A user can create a new Page and mark it as a Skill through the same workflow.
- [ ] A marked Skill Page remains visible and movable in the normal Page tree.
- [ ] A user can unmark a Skill while preserving the underlying Page and content.
- [ ] Skill metadata stores input scope, output mode, status, allowed Tools, approval policy, and editor-menu visibility independently from Page formatting.
- [ ] Invalid ownership and duplicate mark operations are handled safely and idempotently.
- [ ] The Page UI clearly indicates Skill status and exposes metadata management.
